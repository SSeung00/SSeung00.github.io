## [Hook] 표로 저장할 수 없다면?

Unit 0부터 Unit 6까지, 한 가지 전제가 있었다.

**상태 공간이 유한하고, 표에 저장할 수 있다.**

Gridworld의 상태는 셀 번호. Cliff Walking의 상태는 좌표. Blackjack의 상태는 패 합계.
모두 정수로 인덱싱할 수 있는 **이산(discrete) 상태** 다.

이제 CartPole을 보자.

막대기가 달린 수레. 수레가 넘어지지 않도록 좌우로 이동시키는 문제.

상태는 4개 연속 변수다.

- 수레의 위치 (연속)
- 수레의 속도 (연속)
- 막대기 각도 (연속)
- 막대기 각속도 (연속)

각 변수가 연속이라는 것은, 상태의 수가 **무한** 하다는 뜻이다.

Q-table에는 상태마다 행이 하나씩 필요하다. **무한한 행의 표는 만들 수 없다.**

Tabular RL의 벽이다.

## [Fail] 이산화하면 되지 않을까?

가장 먼저 떠오르는 해결책: **연속 상태를 구간으로 나눠 이산화하자.**

각 변수를 100개 구간으로 나눈다면?

상태의 수 = 100 × 100 × 100 × 100 = **1억**

Q-table에는 행동이 2개(좌/우)이므로 셀 수는 2억.
메모리만 수 기가바이트.

그리고 이건 시작일 뿐이다.

- 해상도를 높일수록 상태 수가 기하급수적으로 증가한다. (차원의 저주)
- 대부분의 상태를 학습 중 한 번도 방문하지 못한다.
- 방문하지 않은 상태의 Q값은 영원히 0으로 남는다.

아타리 게임을 생각해보자. 화면 픽셀이 상태다.
84×84 픽셀, 각 픽셀이 256개 값이면 상태 수는 256^(84×84).
우주의 원자 수보다 많다.

**이산화는 해결책이 아니다.**

**"직접 조종" 탭** 에서 CartPole을 직접 경험해보자.

탐구해볼 것들:

- 막대기를 세우는 것이 얼마나 어려운가? 인간은 어떤 "상태"를 보고 판단하는가?

- **"자동 에이전트" 탭** 으로 이동해보자.
  선형 정책이 막대기를 어느 정도 유지하는가?
  이 에이전트는 학습한 것이 아니라 수작업으로 설계된 규칙이다.

- **"왜 Q-table이 불가능한가" 섹션** 을 확인하자.
  이산화 해상도를 높일 때 테이블 크기가 어떻게 폭발적으로 증가하는지 직접 보자.

## [Idea] 표 대신 함수

문제를 다시 보자.

Q-table은 (상태, 행동) 쌍을 입력으로 받아 Q값을 **조회** 한다.
본질적으로 **입력 → 출력 매핑** 이다.

이 매핑을 함수로 나타낼 수 없을까?

$$Q(s, a) \approx f_\theta(s, a)$$

$f_\theta$ 는 파라미터 $\theta$ 를 가진 함수. 상태와 행동을 입력으로 받아 Q값을 출력한다.

표는 방문한 상태만 기억한다. 함수는 **일반화** 한다.
비슷한 상태라면 비슷한 Q값을 출력할 수 있다.

가장 강력한 함수 근사기는 **신경망** 이다.

연속 4차원 상태 벡터를 입력으로 받아, 각 행동의 Q값을 직접 출력한다.
Q-table이 수억 행의 표였다면, 신경망은 수백만 개의 파라미터로 같은 매핑을 근사한다.

이것이 **DQN (Deep Q-Network)** 이다.

Q-Learning의 업데이트 원리는 동일하다.
달라진 것은 Q값을 저장하는 방식: 표 → 신경망.

## [Form] 왜 Tabular가 실패하는가 — 수식으로

### 1. 차원의 저주 (Curse of Dimensionality)

상태 변수가 $d$ 개이고, 각 변수를 $k$ 개 구간으로 이산화하면:

$$|\mathcal{S}| = k^d$$

CartPole: $d = 4$, $k = 100$ → $|\mathcal{S}| = 10^8$

$k$ 를 두 배로 늘리면 상태 수는 $2^d$ 배가 된다.
차원 $d$ 가 크면 이산화 해상도 개선이 불가능해진다.

---

### 2. 함수 근사 (Function Approximation)

Q-table 대신 파라미터 $\theta$ 로 표현되는 함수를 사용한다:

$$\hat{Q}(s, a; \theta) \approx Q^*(s, a)$$

업데이트: TD 오차를 이용해 $\theta$ 를 경사 하강법으로 조정한다.

$$\theta \leftarrow \theta + \alpha \underbrace{\bigl[R + \gamma \max_{a'}\hat{Q}(s', a'; \theta) - \hat{Q}(s, a; \theta)\bigr]}_{\text{TD 오차}} \nabla_\theta \hat{Q}(s, a; \theta)$$

$\nabla_\theta \hat{Q}$ 는 파라미터에 대한 그래디언트 — 신경망에서는 역전파로 계산된다.

---

### 3. Tabular RL vs 함수 근사 비교

| | Tabular RL | 함수 근사 (DQN) |
|---|---|---|
| **상태 표현** | 이산 인덱스 | 연속 벡터 |
| **저장 방식** | 표 (상태 수 × 행동 수) | 신경망 파라미터 |
| **일반화** | ✗ (방문 상태만) | ✓ (유사 상태 공유) |
| **상태 공간** | 유한 필요 | 연속 가능 |
| **수렴 보장** | ✓ (이론적) | △ (불안정 가능) |

함수 근사는 강력하지만 안정성 문제가 생긴다.
신경망 업데이트가 서로 간섭하거나, 발산할 수 있다.

DQN은 이 문제를 두 가지 기법으로 해결한다:
- **Experience Replay**: 과거 경험을 버퍼에 저장, 랜덤 샘플로 학습
- **Target Network**: 업데이트 목표를 주기적으로만 갱신

<details>
<summary>더 알고 싶다면 — DQN의 수렴 문제와 해결책</summary>

순수 Q-Learning에서 신경망 파라미터 $\theta$ 를 업데이트하면 두 가지 불안정성이 생긴다.

**① 연속적 경험의 상관관계**
온라인 학습에서 연속한 경험 $(s_t, a_t, r_t, s_{t+1})$ 은 매우 유사하다.
같은 상태 부근의 데이터만 반복 학습하면 편향이 생긴다.
**해결:** Experience Replay — 과거 경험 $(s, a, r, s')$ 을 버퍼에 저장하고 랜덤 미니배치로 학습.

**② 업데이트 목표 이동 (Moving Target)**
TD 목표 $R + \gamma \max_{a'}\hat{Q}(s', a'; \theta)$ 도 $\theta$ 에 의존한다.
$\theta$ 가 바뀌면 목표도 함께 바뀌어 불안정해진다.
**해결:** Target Network $\theta^-$ — 주 네트워크와 별도로 주기적으로만 동기화되는 네트워크를 목표 계산에 사용.

$$\text{TD 오차} = R + \gamma \max_{a'}\hat{Q}(s', a'; \theta^-) - \hat{Q}(s, a; \theta)$$

## [Next] Part II로

Unit 0부터 Unit 7까지, Tabular RL의 전체 지형을 걸었다.

| 구분 | 방법 | 핵심 질문 |
|---|---|---|
| 탐험 | Bandit | 언제 새것을 시도하는가? |
| 예측 | DP, MC, TD | 가치를 어떻게 추정하는가? |
| 제어 | SARSA, Q-Learning | 어떻게 더 나은 정책을 찾는가? |
| 스펙트럼 | n-step, Dyna-Q | 얼마나 앞을 봐야 하는가? |
| 안전 | Cliff Walking | 탐험 실수를 어떻게 다루는가? |
| **한계** | **CartPole** | **연속 상태를 어떻게 다루는가?** |

Tabular RL은 완전한 이론적 기반이 있다.
수렴이 보장되고, 행동이 설명 가능하다.

하지만 현실 문제에는 통하지 않는다.

**Part II** 는 이 한계를 넘는다.

- **DQN**: Q-Learning + 신경망 — 아타리 게임
- **Policy Gradient**: 가치 함수 없이 정책을 직접 최적화
- **Actor-Critic**: 가치 추정 + 정책 최적화 결합
- **PPO, SAC**: 현대 RL의 표준 알고리즘

Part II에서는 이 모든 것이 **왜** 필요한지, Tabular에서의 직관이 어떻게 연결되는지를 따라간다.
