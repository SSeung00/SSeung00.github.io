## [Hook] 에피소드가 끝나길 기다려야 할까?

Unit 3에서 Monte Carlo를 배웠다.

경험의 평균으로 가치를 추정하는 방법. 모델이 없어도 된다.
블랙잭처럼 에피소드가 짧은 문제에서 잘 작동한다.

하지만 Unit 3 마지막에 이런 질문을 남겼다.

**에피소드가 매우 길거나, 아예 끝이 없다면?**

체스 한 판이 200 수가 걸린다고 하자.
MC는 200 수를 모두 두고 결과(승/패)가 나온 뒤에야 첫 수의 가치를 업데이트한다.

그 200 수 동안 아무것도 배우지 않는다.

더 근본적인 문제도 있다.
MC의 업데이트 수식을 다시 보자:

$$V(s) \leftarrow V(s) + \alpha\bigl[\underbrace{G}_{\text{에피소드 끝까지 기다려야 앎}} - V(s)\bigr]$$

$G$ 를 알려면 에피소드가 끝나야 한다. 이것이 MC의 구조적 제약이다.

**한 발짝만 내딛고 바로 업데이트할 수는 없을까?**

## [Fail] MC로 긴 경로를 학습하면

아래 Gridworld를 보자. 4×5 격자, 구덩이 두 개가 있다.

목표까지의 최단 경로는 7 step이다.
MC로 학습하면 매 에피소드마다 목표에 도달하거나 구덩이에 빠질 때까지 기다려야 한다.
초반에는 목표를 잘 못 찾고, 제자리를 맴도는 에피소드가 많다 — 300 step 제한까지 가는 경우도 생긴다.

그 긴 여정 동안 방문한 상태들의 가치 추정은 매우 부정확하다.
에피소드 하나의 결과(예: −150)로 50개 상태를 일괄 업데이트하면 분산이 크다.

**더 빠르게 배울 방법이 필요하다.**

## [Idea] 추측으로 추측을 수정한다

핵심 아이디어는 이것이다.

> 에피소드 끝까지 기다리지 말고,
> **지금 받은 보상 + 다음 상태의 현재 추정값** 으로 바로 업데이트하자.

$$\underbrace{V(s)}_{\text{현재 추정}} \leftarrow V(s) + \alpha\!\left[\underbrace{R + \gamma V(s')}_{\text{TD 목표}} - V(s)\right]$$

$G$ (실제 리턴) 대신 $R + \gamma V(s')$ (1-step 추정값)을 사용한다.
이 업데이트는 **한 번의 전이** $(s, R, s')$ 만 있으면 바로 실행할 수 있다.

이것이 **TD(0)** 이다. (0은 "0 step 앞을 부트스트래핑한다"는 뜻)

$\bigl[R + \gamma V(s') - V(s)\bigr]$ 를 **TD 오차(TD error)** 라고 부른다.
현재 추정값 $V(s)$ 와 더 나은 추정값 $R + \gamma V(s')$ 의 차이다.

이 오차를 줄이는 방향으로 $V(s)$ 를 조금씩 수정한다.

---

### 비유: 운전 예측

서울에서 부산까지 운전한다.
출발 전에 "4시간 걸릴 것"이라고 예측했다.

MC라면: 부산에 도착한 뒤 "실제로 5시간 걸렸다"고 기록하고 다음 여행 예측을 수정한다.

TD라면: 대전 톨게이트를 지나며 "여기서 부산까지 보통 3.5시간인데, 이미 2시간 왔으니 총 5.5시간"이라고 **중간에 예측을 수정** 한다.

도착하지 않아도 계속 더 나은 추정이 이루어진다.

## [See] SARSA vs Q-Learning을 직접 비교하자

TD는 제어(Control) 문제에 적용하면 두 가지 알고리즘이 된다.

**SARSA (On-policy)** 와 **Q-Learning (Off-policy)** — 수식 한 줄 차이지만 행동이 다르다.

아래 시뮬레이터에서 직접 비교해보자.

{{< simulator src="/simulator/unit-4-td/td-gridworld.html" height="540px" title="Simulator — SARSA / Q-Learning (4×5 Gridworld)" >}}

**구조:** 4×5 격자, 시작(0,0), 목표(3,4), 구덩이 2개(−10 보상, 에피소드 종료).

탐구해볼 것들:

**Q-Learning 탭:**
- **▶ 실행** 을 눌러 100 에피소드 이상 학습시켜보자.
- 화살표 패턴이 목표를 향해 수렴하는가?
- 구덩이 근처 셀의 화살표가 구덩이를 어떻게 피하는지 확인하자.
- **α(학습률)** 을 0.5로 높이면 수렴이 더 빠른가, 더 불안정한가?

**SARSA 탭으로 전환:**
- 같은 파라미터로 학습시키고 Q-Learning과 화살표 패턴을 비교하자.
- **ε(탐험율)** 이 높을 때 (0.3 이상) SARSA는 구덩이를 더 보수적으로 피하는 경향이 있다.
  왜 그럴까? → SARSA는 탐험까지 포함한 정책의 가치를 학습하기 때문이다.

**γ(할인율) 실험:**
- γ = 0.5: 미래를 크게 할인 → 수렴은 빠르지만 먼 목표의 영향이 약해진다.
- γ = 0.99: 먼 보상도 충분히 반영 → 더 긴 경로 계획이 가능하다.

## [Form] 수식으로 압축

### 1. TD(0) — 예측 (Prediction)

임의의 정책 $\pi$ 의 가치 함수를 추정한다.

$$\boxed{V(s) \leftarrow V(s) + \alpha\bigl[R + \gamma V(s') - V(s)\bigr]}$$

```
초기화: V(s) = 0  for all s

에피소드마다:
  s ← 시작 상태

  while s가 종료 상태가 아닐 동안:
    a ← π(s)로 행동 선택
    s', R ← 환경에서 (다음 상태, 보상) 수신
    V(s) ← V(s) + α[R + γV(s') - V(s)]   ← TD(0) 업데이트
    s ← s'
```

에피소드가 끝나길 기다릴 필요 없이, **매 step마다** 업데이트한다.

---

### 2. SARSA — On-policy TD Control

상태 가치 $V(s)$ 대신 **행동 가치 $Q(s,a)$** 를 학습한다.
다음 행동 $A'$ 을 **현재 정책(ε-greedy)으로 선택** 해서 업데이트에 사용한다.

이름 **SARSA** 는 업데이트에 사용하는 다섯 가지 요소에서 왔다:
$$S_t,\ A_t,\ R_{t+1},\ S_{t+1},\ A_{t+1}$$

$$\boxed{Q(s,a) \leftarrow Q(s,a) + \alpha\bigl[R + \gamma\, Q(s', a') - Q(s,a)\bigr]}$$

여기서 $a'$ 는 **ε-greedy 정책으로 선택한 다음 행동** 이다.

```
초기화: Q(s,a) = 0  for all s, a

에피소드마다:
  s ← 시작 상태
  a ← ε-greedy(Q, s)   ← 현재 정책으로 행동 선택

  while s가 종료 상태가 아닐 동안:
    s', R ← 행동 a 실행
    a' ← ε-greedy(Q, s')          ← 다음 행동도 현재 정책으로
    Q(s,a) ← Q(s,a) + α[R + γQ(s',a') - Q(s,a)]
    s ← s';  a ← a'
```

**On-policy**: 탐험(ε-greedy)을 포함한 실제 행동 정책의 가치를 학습한다.
ε이 클수록 구덩이를 "조심스럽게" 피하는 경향이 있다 — 탐험 중 구덩이에 빠질 가능성을 고려하기 때문이다.

---

### 3. Q-Learning — Off-policy TD Control

SARSA와 수식이 거의 같지만, $a'$ 를 선택하는 방식이 다르다.
다음 상태에서 **가장 좋은 행동의 Q값** (greedy)을 사용한다 — 실제로 그 행동을 할지 여부와 무관하게.

$$\boxed{Q(s,a) \leftarrow Q(s,a) + \alpha\bigl[R + \gamma \max_{a'} Q(s', a') - Q(s,a)\bigr]}$$

```
초기화: Q(s,a) = 0  for all s, a

에피소드마다:
  s ← 시작 상태

  while s가 종료 상태가 아닐 동안:
    a ← ε-greedy(Q, s)             ← 탐험을 위해 ε-greedy로 행동
    s', R ← 행동 a 실행
    Q(s,a) ← Q(s,a) + α[R + γ·max_a' Q(s',a') - Q(s,a)]  ← greedy 목표
    s ← s'
```

**Off-policy**: 탐험(ε-greedy)으로 행동하면서도, **최적 정책(greedy)** 의 가치를 학습한다.
행동 정책과 목표 정책이 분리된다.

---

### 4. SARSA vs Q-Learning 비교

| | SARSA | Q-Learning |
|---|---|---|
| **업데이트 목표** | $R + \gamma Q(s', \varepsilon\text{-greedy}(s'))$ | $R + \gamma \max_{a'} Q(s', a')$ |
| **정책 종류** | On-policy | Off-policy |
| **무엇을 학습하나** | 탐험 포함 정책의 가치 | 최적 정책의 가치 |
| **안전성** | 구덩이를 더 보수적으로 피함 | 최적 경로를 추구 (위험할 수 있음) |
| **수렴** | ε → 0 이면 최적으로 수렴 | α 적절하면 최적으로 수렴 |

**핵심 차이 — 절벽 예시 (Cliff Walking):**

Sutton & Barto의 유명한 예제에서, 절벽 바로 옆 최단 경로를 걷는 것이 최적이다.
- Q-Learning은 최적 경로(절벽 옆)를 학습하지만, 탐험 중 가끔 절벽에 떨어진다.
- SARSA는 탐험 위험을 감안해 절벽에서 멀리 우회하는 더 안전한 경로를 학습한다.

### 5. MC vs TD 비교

| | Monte Carlo | TD(0) |
|---|---|---|
| **업데이트 시점** | 에피소드 종료 후 | 매 step |
| **업데이트 목표** | 실제 리턴 $G$ | 추정값 $R + \gamma V(s')$ |
| **부트스트래핑** | ✗ | ✓ |
| **편향(Bias)** | 없음 | 있음 (초기 추정값에 의존) |
| **분산(Variance)** | 높음 | 낮음 |
| **연속 태스크** | ✗ 불가 | ✓ 가능 |
| **에피소드 길이** | 짧을수록 유리 | 관계없음 |

**편향-분산 트레이드오프:**
MC는 불편추정량이지만 에피소드마다 리턴이 크게 달라 분산이 높다.
TD는 편향이 있지만 매 step 업데이트로 분산이 낮다.
이 둘을 연속적으로 이어주는 것이 **TD(λ)** 와 **n-step TD** 다.

<details>
<summary>더 알고 싶다면 — n-step TD와 TD(λ)</summary>

MC와 TD(0)은 양 극단이다.

- **MC**: 에피소드 전체 ($n = \infty$) 를 보고 업데이트
- **TD(0)**: 1 step만 보고 업데이트

그 사이 어딘가가 최적일 수 있다. 이것이 **n-step TD** 다.

$$G_t^{(n)} = R_{t+1} + \gamma R_{t+2} + \cdots + \gamma^{n-1} R_{t+n} + \gamma^n V(S_{t+n})$$

$$V(s) \leftarrow V(s) + \alpha\bigl[G_t^{(n)} - V(s)\bigr]$$

n = 1이면 TD(0), n = ∞이면 MC.

**TD(λ)** 는 모든 n에 대한 n-step 리턴을 지수 가중 평균으로 섞는다.
$\lambda \in [0,1]$ 이 MC(λ=1)와 TD(0)(λ=0) 사이를 조절하는 파라미터다.

$$G_t^\lambda = (1-\lambda)\sum_{n=1}^{\infty} \lambda^{n-1} G_t^{(n)}$$

실용적으로는 **Eligibility Traces** (적격 흔적)으로 효율적으로 구현한다.
과거에 방문한 상태에 흔적(trace)을 남겨두고, 보상이 들어오면 흔적에 비례해 역전파한다.

</details>

## [Next] 다음 질문

Q-Learning으로 Gridworld를 풀었다.
상태가 20개($4 \times 5$), 행동이 4개 — Q-table 크기는 80이다.

그런데 현실 문제를 보자.

아타리 게임의 상태는 화면 픽셀이다. $84 \times 84 \times 4$ 프레임이면 상태 공간이 $256^{84 \times 84 \times 4}$ 에 달한다.
이 상태 공간을 표로 저장할 수 없다.

**Q-table을 함수로 근사하면 어떨까?**

$$Q(s,a) \approx Q(s,a\,;\,\boldsymbol{\theta})$$

$\boldsymbol{\theta}$ 는 신경망의 가중치다.
TD 오차를 손실 함수로 삼아 역전파로 $\boldsymbol{\theta}$ 를 업데이트한다.

$$\mathcal{L}(\boldsymbol{\theta}) = \bigl[R + \gamma \max_{a'} Q(s',a'\,;\,\boldsymbol{\theta}^-) - Q(s,a\,;\,\boldsymbol{\theta})\bigr]^2$$

이것이 **Deep Q-Network (DQN)** 이다.
2015년 DeepMind가 아타리 게임 49개를 인간 수준으로 플레이해 세상을 놀라게 했다.

다음 Unit에서는 함수 근사와 DQN의 핵심 아이디어를 배운다.
