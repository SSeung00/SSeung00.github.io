## [Hook] 규칙을 모른다면?

Unit 2에서 최적 정책을 찾는 방법을 배웠다.
Policy Iteration, Value Iteration — 둘 다 강력했다.

그런데 한 가지 전제가 있었다.

**MDP 모델을 알아야 한다.** 전이 확률 $P(s'|s,a)$ 와 보상 함수 $R(s,a)$.

Gridworld에서는 괜찮았다. 벽이 어디 있는지, 각 방향으로 이동하면 어디로 가는지 — 규칙이 명확했다.

하지만 블랙잭을 생각해보자.

카드는 무작위로 섞인다. 딜러는 숨겨진 규칙으로 행동한다.
어떤 패에서 히트하면 버스트될 확률이 얼마나 되는지, 처음부터 계산하기 어렵다.

**경험으로 배워야 한다.**
직접 게임을 수천 번 플레이하고, 어떤 상태에서 어떤 결과가 나왔는지 기록한다.

이것이 **Monte Carlo** 방법의 핵심이다.

## [Idea] 에피소드를 돌아보면 가치를 추정할 수 있다

핵심 아이디어는 간단하다.

게임 한 판(에피소드)이 끝나면, 그 결과를 역으로 돌아본다.

예를 들어 다음 에피소드를 생각해보자:

```
상태 (17, 딜러6) → 행동: Stick → 상태 (18, 딜러6) → ... → 결과: +1 (승)
```

이 에피소드에서 방문한 각 상태에, 최종 보상 +1을 기록한다.
수천 번 반복하면 각 상태에서의 **평균 리턴** 이 그 상태의 가치가 된다.

$$V(s) \approx \frac{1}{N(s)} \sum_{i=1}^{N(s)} G_i(s)$$

여기서 $G_i(s)$ 는 $i$ 번째 에피소드에서 상태 $s$ 를 방문했을 때의 리턴(최종 보상)이다.

이것이 **Monte Carlo 정책 평가** 다.

Bellman 방정식도 필요 없고, 전이 확률도 필요 없다.
그냥 **경험의 평균** 이다.

블랙잭에서 리턴은 에피소드 끝의 보상뿐이다 (중간 보상이 없다).
따라서 $G = R_{\text{final}} \in \{-1, 0, +1\}$ 이고, $\gamma = 1$ 로 설정한다.

## [Form] 수식으로 압축

### Monte Carlo 정책 평가

**First-Visit MC:**
에피소드에서 상태 $s$ 를 처음 방문했을 때만 리턴을 기록한다.

```
초기화: V(s) = 0, Returns(s) = [] for all s

반복 (에피소드 생성):
  정책 π로 에피소드 생성: S₀, A₀, R₁, S₁, A₁, R₂, ..., Sₜ
  G ← 에피소드의 총 리턴
  for each 에피소드에서 처음 등장하는 상태 s:
    Returns(s).append(G)
    V(s) ← mean(Returns(s))
```

**Every-Visit MC:** 방문할 때마다 기록한다. 결과는 같지만 수렴 속도가 다를 수 있다.

### DP와 MC의 비교

| | Dynamic Programming | Monte Carlo |
|---|---|---|
| **모델 필요** | ✓ ($P$, $R$ 필요) | ✗ (경험만 있으면 됨) |
| **업데이트 시점** | 매 step | 에피소드 종료 후 |
| **부트스트래핑** | ✓ (다음 상태 가치 사용) | ✗ (실제 리턴 사용) |
| **편향** | 없음 | 없음 |
| **분산** | 낮음 | 높음 (에피소드가 짧을수록 낮음) |

**핵심 차이 — 부트스트래핑:**
DP는 $V(s) = R + \gamma V(s')$ 처럼 현재의 추정값 $V(s')$ 를 사용한다.
MC는 실제 에피소드가 끝날 때까지 기다린 뒤, 실제로 얻은 리턴 $G$ 를 사용한다.

MC의 추정값은 불편추정량(unbiased)이다. 에피소드가 많을수록 정확해진다.

### 수렴 보장

대수의 법칙에 의해, 각 상태를 충분히 많이 방문하면:

$$V(s) \xrightarrow{N(s) \to \infty} V_\pi(s)$$

단, 모든 상태가 충분히 방문되어야 한다.
방문 빈도가 낮은 상태(희귀한 패 조합)는 수렴이 느리다.

<details>
<summary>더 알고 싶다면 — MC Control과 탐험 문제</summary>

가치를 **추정** 하는 것과 **최적 정책을 찾는** 것은 다르다.

MC로 정책을 개선하려면 행동 가치 함수 $Q(s, a)$ 를 추정해야 한다.
(상태 가치만으로는 어떤 행동이 최선인지 알 수 없다 — 모델이 없으니까.)

$$Q(s, a) \approx \frac{1}{N(s,a)} \sum G(s, a)$$

그런데 문제가 있다. 결정론적 정책을 따르면 일부 (s, a) 쌍은 **영원히 방문되지 않는다.**

해결책:
1. **Exploring Starts:** 모든 (s, a) 에서 랜덤하게 시작 (이론적 해법)
2. **ε-Greedy:** 항상 ε 확률로 탐험 (실용적 해법)

이 탐험-이용 균형은 Unit 0의 슬롯머신 문제와 동일한 구조다.

## [Next] 다음 질문

Monte Carlo는 강력하지만 한 가지 약점이 있다.

**에피소드가 끝나야 업데이트할 수 있다.**

블랙잭은 에피소드가 짧다. 괜찮다.
하지만 체스, 바둑, 또는 끝이 없는 연속적 작업은?

에피소드가 매우 길거나, 아예 끝이 없는 경우에는
MC처럼 끝까지 기다릴 수 없다.

**한 step만 실행하고 바로 업데이트할 수는 없을까?**

DP처럼 부트스트래핑을 쓰되, 모델은 필요 없는 방법.
MC처럼 실제 경험을 쓰되, 에피소드 끝을 기다리지 않는 방법.

이 두 아이디어의 교차점이 **Temporal Difference (TD) 학습** 이다.
