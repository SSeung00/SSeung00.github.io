## [Hook] 규칙을 모른다면?

Unit 2에서 최적 정책을 찾는 방법을 배웠다.
Policy Iteration, Value Iteration — 둘 다 강력했다.

그런데 한 가지 전제가 있었다.

**MDP 모델을 알아야 한다.** 전이 확률 $P(s'|s,a)$ 와 보상 함수 $R(s,a)$.

Gridworld에서는 괜찮았다. 벽이 어디 있는지, 각 방향으로 이동하면 어디로 가는지 — 규칙이 명확했다.

하지만 블랙잭을 생각해보자.

카드는 무작위로 섞인다. 딜러는 숨겨진 규칙으로 행동한다.
어떤 패에서 히트하면 버스트될 확률이 얼마나 되는지, 처음부터 계산하기 어렵다.

**경험으로 배워야 한다.**
직접 게임을 수천 번 플레이하고, 어떤 상태에서 어떤 결과가 나왔는지 기록한다.

이것이 **Monte Carlo** 방법의 핵심이다.

## [Idea] 에피소드를 돌아보면 가치를 추정할 수 있다

핵심 아이디어는 간단하다.

게임 한 판(에피소드)이 끝나면, 그 결과를 역으로 돌아본다.

예를 들어 다음 에피소드를 생각해보자:

```
상태 (14, 딜러6) → Hit → 상태 (19, 딜러6) → Stick → 결과: +1 (승)
```

이 에피소드에서 방문한 각 상태 — $(14, 딜러6)$ 과 $(19, 딜러6)$ — 에 최종 보상 $+1$ 을 기록한다.
수천 번 반복하면 각 상태에서의 **평균 리턴** 이 그 상태의 가치가 된다.

$$V(s) \approx \frac{1}{N(s)} \sum_{i=1}^{N(s)} G_i(s)$$

여기서 $G_i(s)$ 는 $i$ 번째 에피소드에서 상태 $s$ 를 방문했을 때의 리턴(최종 보상)이다.

이것이 **Monte Carlo 정책 평가** 다.

Bellman 방정식도 필요 없고, 전이 확률도 필요 없다.
그냥 **경험의 평균** 이다.

블랙잭에서 리턴은 에피소드 끝의 보상뿐이다 (중간 보상이 없다).
따라서 $G = R_{\text{final}} \in \{-1, 0, +1\}$ 이고, $\gamma = 1$ 로 설정한다.

## [Form] 수식으로 압축

### 1. Monte Carlo 정책 평가 (Prediction)

**First-Visit MC:**
에피소드에서 상태 $s$ 를 **처음 방문했을 때만** 리턴을 기록한다.

```
초기화: V(s) = 0, N(s) = 0  for all s

반복 (에피소드 생성):
  정책 π로 에피소드 생성: S₀, A₀, R₁, S₁, A₁, R₂, ..., Sₜ
  G ← 에피소드의 총 리턴

  for each 에피소드에서 처음 등장하는 상태 s:
    N(s) ← N(s) + 1
    V(s) ← V(s) + (G - V(s)) / N(s)   ← 증분 평균
```

**Every-Visit MC:** 매 방문마다 기록한다. 방문 횟수가 적은 상태에서 더 빠르게 데이터가 쌓이는 장점이 있지만, 같은 에피소드 내 상관된 샘플이 섞인다.

#### 증분 평균 업데이트

리스트에 모든 리턴을 저장하지 않아도 된다.
평균을 **증분(incremental)** 으로 유지할 수 있다.

$$V(s) \leftarrow V(s) + \frac{1}{N(s)}\bigl[G - V(s)\bigr]$$

여기서 $\frac{1}{N(s)}$ 가 학습률(step size) $\alpha$ 의 역할을 한다.

이를 상수 $\alpha$ 로 고정하면 — **비정상(non-stationary) 문제** 에서 오래된 경험을 잊는 방식이 된다:

$$V(s) \leftarrow V(s) + \alpha\bigl[G - V(s)\bigr]$$

이 형태가 바로 다음 Unit에서 배울 **TD 업데이트의 원형** 이다.

### 2. DP vs MC 비교

| | Dynamic Programming | Monte Carlo |
|---|---|---|
| **모델 필요** | ✓ ($P$, $R$ 필요) | ✗ (샘플 에피소드만 필요) |
| **업데이트 시점** | 매 step | 에피소드 종료 후 |
| **부트스트래핑** | ✓ (추정값 $V(s')$ 사용) | ✗ (실제 리턴 $G$ 사용) |
| **편향(Bias)** | 없음 | 없음 |
| **분산(Variance)** | 낮음 | 높음 |
| **에피소드 필요** | ✗ (연속 태스크 가능) | ✓ (반드시 종료해야 함) |

**핵심 차이 — 부트스트래핑:**

DP는 $V(s) \leftarrow R + \gamma V(s')$ 처럼 **현재의 추정값** $V(s')$ 를 사용한다.
틀릴 수 있는 추정값에 의존하므로 편향이 생길 수 있다(초기에는).

MC는 에피소드가 끝날 때까지 기다린 뒤 **실제로 얻은 리턴 $G$** 를 사용한다.
추정값에 의존하지 않으므로 불편추정량(unbiased)이다.
단, 에피소드 하나하나의 결과가 다르기 때문에 분산이 크다.

### 3. MC Control — 최적 정책 찾기

가치를 **추정** 하는 것(Prediction)과 **최적 정책을 찾는** 것(Control)은 다르다.

그런데 MC로 정책을 개선하려면 한 가지 문제가 있다.

**모델이 없으면 $V(s)$ 만으로는 최적 행동을 알 수 없다.**

DP에서는 $\pi'(s) = \arg\max_a [R(s,a) + \gamma V(s')]$ 처럼 전이 확률을 알기 때문에 $V$ 에서 직접 행동을 뽑을 수 있었다.
모델이 없으면 $s'$ 이 뭔지 모르므로, 대신 **행동 가치 함수** $Q(s,a)$ 를 추정해야 한다.

$$Q(s,a) \approx \frac{1}{N(s,a)} \sum G(s,a)$$

$Q(s,a)$ 를 알면 모델 없이도 그리디하게 행동을 선택할 수 있다:
$$\pi(s) = \arg\max_a Q(s,a)$$

#### 탐험 문제 (Exploration Problem)

결정론적 정책을 따르면 **한 번도 선택하지 않은 행동은 영원히 방문되지 않는다.**
$Q(s,a)$ 를 추정하려면 모든 $(s,a)$ 쌍을 충분히 방문해야 하는데, 이것이 막힌다.

두 가지 해결책이 있다:

**① Exploring Starts (이론적 해법)**
각 에피소드를 임의의 $(s,a)$ 쌍에서 시작해 모든 쌍이 방문되도록 강제한다.
슬롯머신처럼 인위적으로 탐험을 시작할 수 있을 때만 쓸 수 있다.

**② ε-Greedy Policy (실용적 해법)**
항상 확률 $\varepsilon$ 로 랜덤하게 행동한다.
$$\pi(a|s) = \begin{cases} 1 - \varepsilon + \frac{\varepsilon}{|A|} & \text{if } a = \arg\max_{a'} Q(s,a') \\ \frac{\varepsilon}{|A|} & \text{otherwise} \end{cases}$$

이 구조는 **Unit 0의 슬롯머신 탐험 문제와 완전히 동일하다.**

#### On-policy MC Control 알고리즘

```
초기화: Q(s,a) = 0, N(s,a) = 0  for all s, a
ε-Greedy 정책 π 초기화

반복:
  현재 π로 에피소드 생성: S₀,A₀,R₁, S₁,A₁,R₂, ..., Sₜ
  G ← 에피소드의 총 리턴

  for each (s,a) 처음 방문하는 쌍:
    N(s,a) ← N(s,a) + 1
    Q(s,a) ← Q(s,a) + (G - Q(s,a)) / N(s,a)

  # Policy Improvement (π를 Q에 대해 ε-Greedy로 갱신)
  for each s in episode:
    a* ← argmax_a Q(s,a)
    π(a|s) ← (1-ε+ε/|A|) if a=a*, else ε/|A|
```

### 4. 수렴 보장

대수의 법칙에 의해, 각 상태를 충분히 많이 방문하면:

$$V(s) \xrightarrow{N(s) \to \infty} V_\pi(s)$$

단, 모든 상태가 충분히 방문되어야 한다.
방문 빈도가 낮은 상태(희귀한 패 조합)는 수렴이 느리다.

ε-Greedy를 쓰면 모든 $(s,a)$ 쌍이 무한히 방문되지만(GLIE 조건),
$\varepsilon$ 이 너무 크면 수렴한 정책도 랜덤 성분이 남는다.
실용적으로는 에피소드가 늘어날수록 $\varepsilon$ 을 점점 줄여가는 방식을 쓴다.

<details>
<summary>더 알고 싶다면 — Off-policy MC와 중요도 샘플링</summary>

On-policy 방법은 **탐험하는 정책과 최적화하는 정책이 동일하다.**
ε-Greedy를 써야 하므로, 수렴한 정책도 완전히 최적이 되지 않는다.

**Off-policy** 방법은 이 둘을 분리한다.

- **행동 정책(Behavior Policy)** $b$: 실제로 환경을 탐험하는 정책 (랜덤 등)
- **목표 정책(Target Policy)** $\pi$: 최적화하려는 정책 (완전 탐욕적 가능)

$b$ 로 수집한 데이터로 $\pi$ 를 평가하려면 **중요도 샘플링(Importance Sampling)** 이 필요하다.

$$V_\pi(s) \approx \frac{\sum_i \rho_i G_i}{\sum_i \rho_i}, \quad \rho_i = \prod_{t} \frac{\pi(A_t|S_t)}{b(A_t|S_t)}$$

$\rho$ 가 누적 곱이라 분산이 폭발적으로 커질 수 있다 — Off-policy MC의 근본적인 단점이다.
이 분산 문제를 단계별 업데이트로 해결한 것이 바로 TD 학습이다.

## [Next] 다음 질문

Monte Carlo는 두 가지를 해결했다.

- **모델 없이도** 가치 함수를 추정할 수 있다.
- **경험의 평균** 이라는 단순한 원리로 동작한다.

하지만 한 가지 약점이 있다.

**에피소드가 끝나야 업데이트할 수 있다.**

블랙잭은 에피소드가 짧다. 괜찮다.
하지만 체스, 바둑, 또는 끝이 없는 연속적 작업은?
에피소드 하나가 수천 step에 달한다면, 매번 끝까지 기다리는 건 비효율적이다.

그리고 수식을 다시 보자.

$$V(s) \leftarrow V(s) + \alpha\bigl[\,\underbrace{G}_{\text{실제 리턴}} - V(s)\bigr]$$

$G$ 는 에피소드 **전체** 를 다 경험한 뒤에야 알 수 있다.

만약 $G$ 대신 **지금 받은 보상 + 다음 상태의 추정값** 을 쓰면 어떨까?

$$V(s) \leftarrow V(s) + \alpha\bigl[\underbrace{R + \gamma V(s')}_{\text{1-step 추정}} - V(s)\bigr]$$

에피소드가 끝나길 기다리지 않고, **한 step만 실행한 뒤 바로 업데이트** 할 수 있다.

DP처럼 부트스트래핑을 쓰되 모델이 필요 없고,
MC처럼 실제 경험을 쓰되 에피소드 끝을 기다리지 않는 방법.

이 두 아이디어의 교차점이 **Temporal Difference (TD) 학습** 이다.

다음 Unit에서는 TD(0)과 Q-Learning을 배운다.
